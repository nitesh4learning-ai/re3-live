1. Overview

2. The Six Stages

Stage 1 POINT — Detection

The user opens the camera. ML Kit processes video frames in real-time on-device.

ComponentDetail
ML Kit Image LabelingReturns labels + confidence scores (e.g., "chocolate" 0.87, "food" 0.95)
ML Kit Barcode ScanningDetects EAN-13, UPC-A, QR codes. Boosts confidence by +0.3 for packaged food.
Frame SamplingEvery 3rd frame processed. Debounce 300ms. Max 5 labels per frame.
ThresholdsConfigured in detection_thresholds.json per category (food: 0.55, animal: 0.6, plant: 0.55, medication: 0.7)

Stage 2 SEE — Classification

Raw ML Kit labels are translated into specific search terms and matched against local data.

ComponentDetail
Label Translationml_label_translation.json maps generic labels to API-specific terms (e.g., "chocolate" → "dark chocolate bar 70% cocoa" for USDA)
Compound Label MergeMultiple ML labels combined: ["food", "chocolate", "dark"] → "dark chocolate bar"
Local DB LookupBarcode → product_cache_seed.json, breed → dog_breeds.json/cat_breeds.json, toxicity → toxic_plants.json
Category Routingaction_routing_rules.json (72 rules) determines the processing chain per detection type

Stage 3 THINK — Intelligence

Multi-tiered intelligence applies scoring, rules, and AI reasoning.

TierSourceCostLatency
Tier 1: Bundled JSONOn-device data files$0<10ms
Tier 2: Rule EngineScore calculation, allergen check, routing rules$0<50ms
Tier 3: Firestore CacheShared product cache (pre-seeded from Open Food Facts)~$0.0001/read100-300ms
Tier 4: External APIsUSDA, Nutritionix, PlantNet, Open Food Facts$0 (free tier)1-5s
Tier 5: Claude HaikuAI reasoning for unknowns, label refinement, recommendations$0.0002/call1-3s

Stage 4 SNAP — Decision

Conflict resolution determines the final recommendation when sources disagree.

ScenarioResolution
Safety flag from ANY sourceSafety always wins. Allergens, toxicity, recalls block all other actions.
Score < 50 but LLM says "buy"Show both perspectives. LLM overrides only if score delta > 20 AND provides justification.
Score ≥ 70 but LLM says "swap"LLM wins (has dietary preference context user's score engine lacks).
Both agreeGreen light (both buy) or swap card (both swap).
Unknown itemLLM is sole decision maker. Prompt retry with different angle.

Full rules: conflict_resolution.json (12 rules with priority ordering)

Every scan makes the system smarter for the next user.

Increment scan_countPopular products bubble up for priority refresh

4. Implemented Data Files

data/snapsense/
├── detection_thresholds.json # ML Kit confidence thresholds per category
├── ml_label_translation.json # ML Kit label → API search term mapping
├── action_routing_rules.json # 72 routing rules (POINT→SEE→SNAP→ACT)
├── deep_link_templates.json # 19 deep link templates for ACT stage
├── conflict_resolution.json # 12 conflict resolution rules
├── api_rate_limit_manager.json # Quota management + fallback chains
├── firestore_product_seeder.json # Cold-start seeding strategy
├── dietary_preferences.json # On-device dietary prefs schema
├── dog_breeds.json # ~400 breeds with ML labels
├── cat_breeds.json # ~70 breeds with ML labels
├── toxic_plants.json # ~1000 toxic plants with symptoms
├── common_foods.json # ~500 foods with nutrition data
└── product_cache_seed.json # ~2000 barcoded products

detection_thresholds.json

Confidence thresholds for ML Kit per category. Food: 0.55, Animal: 0.6, Plant: 0.55, Medication: 0.7. Includes barcode config, performance limits, and sub-category granularity.

ml_label_translation.json

Bridges generic ML Kit labels to API-specific search terms. 25+ food mappings with modifiers. Compound label merge rules (20 rules). Claude Haiku fallback for unknown labels ($0.0002/call).

conflict_resolution.json

12 prioritized rules for when on-device score engine disagrees with Claude LLM. Safety always wins (priority 0). Conditional resolution with "show both perspectives" UI pattern.

api_rate_limit_manager.json

Manages 6 API providers' free-tier quotas. Time-weighted budget distribution across day. Fallback chains per use case. Degraded mode UI when quotas exhausted.

firestore_product_seeder.json

3-phase seeding: bundled 2K products (day 0), Firestore 100K from Open Food Facts dump (pre-launch), user flywheel (ongoing). Health score computation algorithm.

dietary_preferences.json

On-device only (Hive). Never synced to cloud. Allergen label mapping (17 allergens × 10+ synonyms each). Household members. Passed to Claude as ephemeral context.

5. Gap Resolutions

Eight architectural gaps were identified and resolved:

Gap 1: Phase 1 JSON Libraries — Never Built

Problem: 6 bundled JSON files were designed but never created (dog breeds, cat breeds, toxic plants, common foods, product cache, routing rules).

Resolution: All 6 files implemented. Dog breeds (~400), cat breeds (~70), toxic plants (~1000), common foods (~500), product cache seed (~2000 barcodes), action routing rules (72 rules).

File: data/snapsense/*.json

Gap 2: Routing Rules — Designed But No Code

Problem: Four config files (detection_thresholds, routing_rules, action_registry, deep_link_templates) were spec'd but never implemented.

Resolution: All 4 files built with full production-ready schemas. detection_thresholds.json (per-category with sub-categories), deep_link_templates.json (19 templates with app scheme + fallback URL).

Gap 3: ML Kit Label → API Search Term Translation

Problem: ML Kit returns generic labels like "chocolate." USDA/Nutritionix expect specific terms like "dark chocolate bar 70% cocoa."

Resolution: ml_label_translation.json implements a 4-level strategy: (1) exact match, (2) label map with modifiers, (3) compound label merge (20 rules), (4) Claude Haiku fallback at $0.0002/call with 30-day caching.

Gap 4: Claude vs Score Conflict Resolution

Problem: Level 2 says score < 50 = "swap" but Level 3 (Claude) says "product_buy." No resolution defined.

Resolution: conflict_resolution.json defines 12 prioritized rules. Safety always wins (P0). Score-vs-LLM conflicts resolved conditionally: LLM overrides only if delta > 20 AND provides justification. Otherwise "show both perspectives" with user choice.

Gap 5: Free API Rate Limits Will Hit Fast

Problem: PlantNet (500/day), UPCitemdb (100/day), Nutritionix (200/day) exhaust by noon at 10K users.

Resolution: api_rate_limit_manager.json implements: time-weighted budget distribution (40% morning, 35% afternoon, 25% evening), fallback chains per use case, graceful degraded mode UI, and scaling plan (1K → 10K → 100K users).

Gap 6: Firestore Product Collection — Not Seeded

Problem: Day 1 with zero cache = every scan hits external APIs, burning free quotas instantly.

Resolution: firestore_product_seeder.json defines 3-phase strategy: Phase 1 (bundled 2K products in app), Phase 2 (pre-seed Firestore with 100K products from Open Food Facts bulk dump at $0.22 total), Phase 3 (user flywheel with 90-day staleness refresh).

Gap 7: Dietary Preferences Storage for MVP

Problem: Allergen warnings require dietary prefs, but Tier 4 (private data) was parked.

Resolution: dietary_preferences.json defines on-device-only schema (Hive, never Firestore). 17 allergens with synonym mappings (10+ variants each). Passed to Claude as ephemeral context. 3-screen onboarding flow. Household members with pet types for plant toxicity warnings.

Gap 8: Design Without Building

Problem: 8 rounds of architecture discussion without producing implementation artifacts.

Resolution: This commit. All 8 gaps addressed with concrete, production-ready JSON configuration files totaling ~4,500+ lines of structured data.

Fallback Chains

Barcode Lookup:   Local Cache → Open Food Facts → UPCitemdb → Claude Haiku
Food Nutrition:   Local Cache → USDA FDC → Nutritionix → Edamam → OFF → Claude Haiku
Plant ID:         Local Cache (toxic_plants.json) → PlantNet → Claude Haiku
Animal ID:        Local Cache (breeds JSON) → Claude Haiku (rare breeds only)

Degraded Mode

When all free APIs are exhausted for the day and Claude Haiku budget is exceeded:

Banner: "Some features are limited today. Results may be less detailed."

Still works: Barcode lookup (local cache), breed identification (local JSON), toxic plant warnings (local JSON), health scores for cached products.

Disabled: Real-time nutrition lookup for new items, PlantNet API identification, label translation for unknown foods.
