## Schema Design

Good schema design makes the difference between reliable and brittle structured outputs.

<SchemaDesigner/>

<ExpandableSection title="Schema Design Tips" icon="💡" defaultOpen>

- Keep schemas flat when possible — deeply nested objects increase failure rates
- Use enums to constrain values — the model can only pick from your options
- Add descriptions to guide the model — "confidence: a number between 0 and 1"
- Make non-critical fields optional — required fields may cause the model to hallucinate values
- Use arrays for variable-length data — let the model decide how many entities to extract

</ExpandableSection>
