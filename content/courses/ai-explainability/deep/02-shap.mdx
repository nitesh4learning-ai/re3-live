## SHAP: SHapley Additive exPlanations

<JargonTip term="SHAP">SHAP</JargonTip> is based on game theory — specifically Shapley values from cooperative game theory. It treats each feature as a "player" and the model prediction as the "payoff," calculating each feature's fair contribution to the outcome.

### How SHAP Works

The core idea: for each prediction, calculate how much each feature contributed to pushing the prediction away from the baseline (average prediction).

<CodeBlock language="python" label="SHAP in Action — Credit Risk Model">
{`import shap
import xgboost as xgb
import pandas as pd

# Train a model (credit risk prediction)
model = xgb.XGBClassifier()
model.fit(X_train, y_train)

# Create SHAP explainer
explainer = shap.TreeExplainer(model)

# Get SHAP values for the entire dataset (global)
shap_values = explainer.shap_values(X_test)

# Global explanation: which features matter most overall?
shap.summary_plot(shap_values, X_test)
# Shows: income, credit_history, debt_ratio are top predictors

# Local explanation: why was THIS applicant rejected?
applicant_idx = 42
shap.waterfall_plot(
    shap.Explanation(
        values=shap_values[applicant_idx],
        base_values=explainer.expected_value,
        data=X_test.iloc[applicant_idx],
        feature_names=X_test.columns.tolist()
    )
)
# Shows: high debt_ratio (+0.3) and short credit_history (+0.2)
# pushed this applicant toward "reject"`}
</CodeBlock>

### Interpreting SHAP Values

<AnalogyBox emoji="⚖️" title="Fair Division of Credit">
Imagine three sales reps close a deal together. How do you fairly divide the commission? Shapley values calculate each person's marginal contribution across all possible team combinations. SHAP does the same for features: how much does each feature contribute to pushing the prediction above or below the baseline, across all possible feature combinations?
</AnalogyBox>

**Positive SHAP value** = this feature pushed the prediction **higher** (toward positive class)
**Negative SHAP value** = this feature pushed the prediction **lower** (toward negative class)
**Magnitude** = how strongly the feature influenced the prediction

### SHAP for LLM Outputs

SHAP was designed for traditional ML. For LLMs, direct SHAP application is computationally infeasible (millions of parameters). Instead, use:

- **Token-level attribution**: Which input tokens most influenced the output?
- **Prompt component analysis**: Which parts of the prompt (system message, context, user query) most affected the response?
- **Chain-of-thought tracing**: Follow the reasoning steps the model took
