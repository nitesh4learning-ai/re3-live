Academic XAI (SHAP plots in Jupyter notebooks) answers data scientist questions. Enterprise XAI answers auditor, regulator, and executive questions. The gap is enormous.

### Agent Observability: Tracing Multi-Step AI Decisions

For agentic AI systems (LangGraph, multi-agent pipelines), explainability means tracing the **full execution chain**: which tools were called, what data was retrieved, what decisions were made, and why.

<CodeBlock language="python" label="LangSmith Agent Tracing">
{`import os

# Enable LangSmith tracing — captures every step
os.environ["LANGCHAIN_TRACING_V2"] = "true"
os.environ["LANGCHAIN_API_KEY"] = "your-key"
os.environ["LANGCHAIN_PROJECT"] = "supplier-onboarding-agent"

# Now every LangGraph node execution is traced:
# - Input state at each node
# - LLM prompts and responses
# - Tool calls and results
# - Routing decisions (which conditional edge was taken)
# - Token usage and latency per node
# - Error traces with full context

# In LangSmith dashboard you can:
# 1. See the full graph execution as a timeline
# 2. Click any node to see its input/output
# 3. Filter by latency, cost, or error rate
# 4. Compare runs across different versions
# 5. Build regression test suites from production traces`}
</CodeBlock>

### Training Data Attribution

The frontier of enterprise XAI: tracing a model's output back to the **specific training data** that shaped it. When a financial model flags a transaction as suspicious, which historical patterns drove that conclusion?

<AnalogyBox emoji="📋" title="Data Lineage Meets AI">
At McKesson, you trace every golden record back to its source systems: which SAP instance, which business unit, which steward approved it. Training data attribution does the same for AI predictions: which training examples most influenced this output? This is the bridge between your data governance expertise and AI governance — the same lineage principles applied to a new domain.
</AnalogyBox>
