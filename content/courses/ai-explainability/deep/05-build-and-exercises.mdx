## Build: Explainability Layer for a Production Model

### Project: Add XAI to a Classification Model

Take a real classification problem (supplier risk scoring, product category prediction, or credit approval) and add a complete explainability layer.

**Deliverables:**
1. **Trained model** with SHAP global explanations (summary plot, feature importance ranking)
2. **LIME local explanations** for 5 individual predictions with business-language narratives
3. **Counterfactual analysis** — for 3 predictions, show what minimal change would flip the outcome
4. **Model card** documenting capabilities, limitations, training data, and known biases
5. **Governance report** mapping your explainability approach to NIST MEASURE and EU AI Act requirements

### Stretch Goals

1. Build an interactive SHAP dashboard (Streamlit or Next.js) that lets stakeholders explore feature importance
2. Add agent execution tracing with LangSmith to an existing LangGraph pipeline
3. Create a "bias audit report" using Fairlearn to test for demographic disparities
4. Design a monitoring dashboard that alerts when model explanations drift significantly from baseline
