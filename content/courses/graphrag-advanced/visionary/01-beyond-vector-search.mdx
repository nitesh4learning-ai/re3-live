## Beyond Vector Search

Vector search is powerful — but it only finds documents *similar* to your query. It cannot follow chains of reasoning, discover multi-hop relationships, or answer questions that require connecting information scattered across many documents. **<JargonTip term="GraphRAG">GraphRAG</JargonTip>** adds a <JargonTip term="knowledge graph">knowledge graph</JargonTip> layer that enables relationship-aware retrieval.

<AnalogyBox emoji="🗺️" title="The map vs the satellite photo">
  A satellite photo (vector search) shows you everything nearby. A map (knowledge graph) shows you how places are connected — roads, borders, rivers. To answer "How do I get from A to B via C?", you need the map.
</AnalogyBox>


### The Fundamental Limitations of Vector-Only RAG

- **No Multi-Hop Reasoning** -- "What did Einstein's colleagues go on to found?" requires: Einstein to colleagues to people to founded to organizations. Vector search finds documents about Einstein but cannot traverse this chain.
- **Lost Structural Context** -- Two documents may discuss the same entity in very different contexts. Vector similarity averages these into noise, losing structural distinctions.
- **No Global Understanding** -- Vector RAG cannot answer "What are the dominant themes across all our documents?" -- it only operates locally around each query.
- **Entity Disambiguation Failure** -- "Apple" the company vs "apple" the fruit -- embedding-based search collapses these; a typed knowledge graph distinguishes them explicitly.


### Knowledge Graphs 101

- **Nodes (Entities)** -- Real-world objects: People, Organizations, Concepts, Events, Products. Each node has a type and properties.
- **Edges (Relationships)** -- Named, directed connections between nodes: WORKS_AT, FOUNDED, DEPENDS_ON, CITED_BY. Edges can also have properties.
- **Properties** -- Attributes on nodes and edges: name, date, confidence score, source document, embedding vector. Rich metadata for filtering.

<CodeBlock language="text" label="Knowledge Graph Structure Example" code={`// Nodes
(n1:Person   {name: "Geoffrey Hinton",  field: "AI"})
(n2:Concept  {name: "Backpropagation", domain: "ML"})
(n3:Org      {name: "Google Brain",    founded: 2011})
(n4:Paper    {name: "ImageNet 2012",   citations: 94000})

// Edges
(n1)-[:DEVELOPED]->(n2)
(n1)-[:WORKED_AT]->(n3)
(n1)-[:AUTHORED]->(n4)
(n4)-[:INTRODUCED]->(n2)

// Multi-hop Query:
// "What concepts did Google Brain researchers develop?"
// n3 <-[:WORKED_AT]- n1 -[:DEVELOPED]-> n2
// Answer: Backpropagation (via Hinton → Google Brain)`} />
