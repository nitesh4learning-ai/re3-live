## Hybrid Retrieval

No single retrieval method dominates all query types. Production GraphRAG systems combine **vector search** (semantic <JargonTip term="embedding">embedding</JargonTip> similarity), **graph traversal** (relationship paths), and **keyword search** (exact matching) in a <JargonTip term="hybrid search">hybrid search</JargonTip> approach, then use intelligent **query routing** and **result reranking** to fuse the best of all three.

### Query Routing: When to Use Which Method

| Signal | Method | Why |
|--------|--------|-----|
| "How are X and Y related?" | Graph traversal | Explicit relationship question -- graph edges answer this directly |
| "What is the main theme across all documents?" | Global search (community summaries) | Dataset-wide synthesis -- requires hierarchical community summaries |
| Contains specific version number, ID, or exact term | BM25 keyword search | Exact match needed -- vector embeddings blur precision |
| "Something about X" or conceptual query | Vector search | Semantic similarity -- no exact terms known |
| Complex multi-part question | All three + rerank | RRF fusion maximizes recall; reranker optimizes precision |

        reranked = self.reranker.rerank(query, fused, top_k=k)

    def _rrf_fusion(self, results_by_method: dict, k: int = 60) -> List[Dict]:
        """Reciprocal Rank Fusion: score = sum(1 / (k + rank_i))"""
        doc_scores = {}

        for method, docs in results_by_method.items():
            for rank, doc in enumerate(docs):
                doc_id = doc['id']
                if doc_id not in doc_scores:
                    doc_scores[doc_id] = {'doc': doc, 'score': 0.0}
                doc_scores[doc_id]['score'] += 1.0 / (k + rank + 1)

        sorted_docs = sorted(doc_scores.values(),
                            key=lambda x: x['score'], reverse=True)
        return [item['doc'] for item in sorted_docs]

            return 'relational'
        if any(c.isdigit() for c in query) or query.count('"') >= 2:
