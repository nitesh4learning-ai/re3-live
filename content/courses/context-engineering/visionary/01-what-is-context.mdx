## What Is Context?

<AnalogyBox emoji={'D83CDFAC'} title="The Director's Cut">
{`Context engineering is like being a movie director — you choose what the camera sees. The model can only reason about what's in its context window. Everything else might as well not exist.`}
</AnalogyBox>

Context is **everything the model sees** when generating a response:

- **System Prompt** -- Instructions that define role, behavior, constraints
- **Retrieved Documents** -- Knowledge fetched via RAG — external memory
- **Conversation History** -- Previous messages — session memory
- **Tool Results** -- Function call outputs — real-time data

The context window is a **fixed-size whiteboard**. Every <JargonTip term="token">token</JargonTip> written on it shapes the response. Once full, something must be removed to add anything new.
