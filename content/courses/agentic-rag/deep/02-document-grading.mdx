## LLM-as-Judge: Grading Retrieval Quality

The core innovation of Agentic RAG is using an LLM to evaluate whether retrieved documents actually answer the user's question. This is the **LLM-as-judge** pattern — the same LLM that generates answers also grades the inputs it receives.

### The Grading Node

<CodeBlock language="python" label="Document Grading Node — LLM Evaluates Relevance" code={`from langchain_anthropic import ChatAnthropic
from langchain_core.messages import SystemMessage, HumanMessage
from typing import TypedDict, List, Literal

class RAGState(TypedDict):
    query: str
    documents: List[dict]        # Retrieved docs with content + metadata
    relevant_docs: List[dict]    # Docs that passed grading
    irrelevant_count: int        # How many docs failed grading
    generation: str              # Final generated answer
    retry_count: int
    hallucination_score: float

llm = ChatAnthropic(model="claude-sonnet-4-20250514", temperature=0)

GRADING_PROMPT = """You are a document relevance grader.
Given a user question and a retrieved document, determine if 
the document contains information relevant to answering the question.

Return ONLY "relevant" or "irrelevant" — nothing else.

User question: {query}

Document content:
{document}
"""

def grade_documents(state: RAGState) -> dict:
    """
    Grade each retrieved document for relevance.
    
    This is where Agentic RAG diverges from Naive RAG.
    Instead of blindly passing all docs to the generator,
    we filter out irrelevant ones first.
    
    In MDM terms: this is the validation step before
    promoting data to the golden record.
    """
    query = state["query"]
    relevant = []
    irrelevant_count = 0
    
    for doc in state["documents"]:
        response = llm.invoke([
            HumanMessage(content=GRADING_PROMPT.format(
                query=query,
                document=doc["content"][:1000]  # Truncate for cost
            ))
        ])
        
        grade = response.content.strip().lower()
        
        if "relevant" in grade and "irrelevant" not in grade:
            relevant.append(doc)
        else:
            irrelevant_count += 1
            print(f"  ❌ Filtered: {doc.get('source', 'unknown')[:50]}")
    
    print(f"  📊 {len(relevant)}/{len(state['documents'])} docs passed grading")
    
    return {
        "relevant_docs": relevant,
        "irrelevant_count": irrelevant_count,
    }`}/>

### Why Grade Individual Documents?

<AnalogyBox emoji="🔬" title="Peer Review in Academia">
Think of document grading like academic peer review. A researcher doesn't cite every paper they find — they evaluate each source for relevance, methodology quality, and applicability to their specific question. Papers that don't meet the bar get excluded. The final paper is stronger because every citation was vetted. Agentic RAG does the same thing with retrieved documents.
</AnalogyBox>
