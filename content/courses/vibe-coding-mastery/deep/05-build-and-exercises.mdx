## Build: Set Up a Production-Grade Vibe Coding Environment

### Project: Configure Claude Code for an Existing Codebase

Take an existing project (SnapSense, Re³, or your own) and create a complete vibe coding infrastructure:

**Deliverables:**
1. **CLAUDE.md** — Comprehensive project memory file covering tech stack, architecture rules, code style, current sprint, and known issues
2. **Skills directory** — At least 3 custom skills (e.g., create-component.md, api-endpoint.md, database-migration.md)
3. **MCP configuration** — Connect at least one MCP server (filesystem + one external service)
4. **Workflow document** — Your personal vibe coding playbook: when to use Plan Mode, thinking levels for different tasks, escape key triggers

### Stretch Goals

1. **CI integration** — Set up Claude Code in GitHub Actions for automated PR review
2. **Team CLAUDE.md** — Create a CLAUDE.md that works for your entire team, not just you
3. **Prompt library** — Document your 10 most effective prompts with examples and expected outputs
4. **Before/after audit** — Take a messy codebase section, document it, then use Claude Code with your new guardrails to refactor it. Compare quality.
