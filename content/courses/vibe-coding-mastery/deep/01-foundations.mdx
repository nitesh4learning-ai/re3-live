## Beyond "Ask and Pray": How Vibe Coding Actually Works

Vibe coding isn't typing a prompt and hoping for the best. It's a **collaboration methodology** — treating Claude Code as a junior developer who types at 200 WPM, has read every library's documentation, but needs clear direction, active review, and architectural guardrails.

<AnalogyBox emoji="🏗️" title="Construction Foreman, Not Magic Wand">
You're the construction foreman. Claude Code is the crew — incredibly fast, never tired, knows every building technique. But without blueprints, specifications, and your active oversight, they'll build a structurally unsound house very quickly. The foreman's job isn't to lay bricks — it's to ensure every brick goes in the right place.
</AnalogyBox>

### The CLAUDE.md File: Your Project's Memory

The single most important file in any vibe coding project. CLAUDE.md is Claude Code's persistent memory — it reads this file at the start of every session to understand your project's rules, patterns, and constraints.

<CodeBlock language="markdown" label="Example CLAUDE.md for a Next.js Project">
{`# Project: SnapSense AI
## Tech Stack
- Flutter 3.x with Dart
- Firebase (Auth, Firestore, Crashlytics)
- Claude Haiku API via Vercel gateway

## Architecture Rules
- All scan history stored in Hive (local only, NEVER Firestore)
- LLM calls go through snapsense-ai-gateway, never direct
- Every new feature needs a corresponding test
- Use Provider pattern for state management

## Code Style
- Dart: follow effective_dart lints
- Max 200 lines per file — split if longer
- Every public method needs a doc comment
- Error handling: never swallow exceptions silently

## Current Sprint
- Working on: Rating System exclusion engine
- Known bug: Tylenol shows "UNKNOWN" instead of shield
- Do NOT modify: Camera pipeline (stable, don't touch)`}
</CodeBlock>

<AnalogyBox emoji="📋" title="CLAUDE.md = Your Data Dictionary">
In MDM, the data dictionary defines field rules, valid values, and business constraints. CLAUDE.md serves the same purpose for your codebase — it tells Claude what patterns to follow, what files to leave alone, and what constraints are non-negotiable. Without it, Claude invents its own patterns every session.
</AnalogyBox>
