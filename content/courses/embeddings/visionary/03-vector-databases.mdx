## Vector Databases

Once you have embeddings, you need somewhere to store and search them efficiently. **Vector databases** are specialized databases optimized for storing high-dimensional vectors and performing fast <JargonTip term="vector search">similarity searches</JargonTip>.

The key challenge: searching millions of vectors by similarity needs to be fast. Exact comparison against every vector is too slow, so vector DBs use **Approximate Nearest Neighbor (ANN)** algorithms.

| Database | Type | Best For |
|----------|------|----------|
| Pinecone | Managed cloud | Simplest setup, serverless scaling |
| Weaviate | Self-hosted or cloud | Hybrid search (vector + keyword) |
| pgvector | PostgreSQL extension | Already using PostgreSQL |
| Chroma | Lightweight/embedded | Prototyping, small datasets |
| Qdrant | Self-hosted or cloud | High performance, filtering |

<ExpandableSection title="How ANN Search Works (HNSW)" icon="⚡">

The most popular ANN algorithm is **HNSW (Hierarchical Navigable Small World)**. Think of it like a skip list for vectors:

- Build a multi-layer graph of vectors, with fewer nodes at higher layers
- Search starts at the top layer (few nodes, big jumps) and drills down
- At each layer, greedily move toward the nearest neighbor

HNSW trades ~5% accuracy for 100x speed improvement. For most applications, the small accuracy loss is negligible.

</ExpandableSection>
