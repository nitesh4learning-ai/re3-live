## MCP Architecture

MCP uses a three-layer architecture: **Host**, **Client**, and **Server**. Understanding these roles is key to building and using MCP systems.

| Role | What It Does | Example |
|------|-------------|---------|
| **Host** | The application the user interacts with | Claude Desktop, VS Code, IDE |
| **Client** | Manages the connection to MCP servers | Built into the host app |
| **Server** | Exposes tools, resources, and prompts | GitHub server, Slack server, DB server |
